SLD/MLD

The Token-Efficient Data Format for LLMs and Streams

Why SLD?

JSON is great, but it's verbose. In the era of Large Language Models (LLMs), every token counts. SLD (Single Line Data) reduces token usage by eliminating structural overhead like quotes, braces, and whitespace, without losing type safety or structure.

JSON

{
  "name": "Alice",
  "age": 30,
  "tags": ["admin", "user"],
  "verified": true
}
~45 tokens

SLD

name[Alice;age[30;tags{admin~user};verified[^1
~18 tokens

Key Features

  • Token Efficient: Uses rare characters (; [ { ^) as delimiters to minimize token splitting.
  • Stream Ready: MLD (Multi Line Data) is the newline-separated variant, perfect for logs and Unix pipes.
  • Type Safe: Supports Strings, Numbers, Booleans (^1/^0), Nulls (^_), and Arrays.
  • Zero Overhead: No quoting required for standard alphanumeric strings.

Syntax at a Glance

Symbol Meaning Example
; Field Separator key1[val1;key2[val2
[ Property Marker age[30
{ } Array Markers colors{red~blue}
~ Array Separator 1~2~3
^ Escape / Special ^1 (true), ^_ (null)
